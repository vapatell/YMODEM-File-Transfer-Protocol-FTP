Part 1: YMODEM Sender Simulation (C++)

This phase of the project focuses on implementing a simplified file transfer protocol simulator based on the YMODEM protocol using C++.

Objective:
Develop a sender-side implementation that simulates batch file transfer without real hardware communication. 
Instead of transmitting over a serial interface, the system writes protocol-compliant byte streams to an output file.

Key Features:

# Protocol Simulation
	
	> Implemented core components of the YMODEM sender workflow, including:
	  File metadata block generation
		- Data block construction (fixed 128-byte payloads)
		- End-of-transmission signaling (EOT sequence)
		- Simulated receiver interactions (ACK/NAK handling)
	
	> Error Detection
	  Integrated 16-bit CRC (Cyclic Redundancy Check) using CRC-16 to ensure data integrity across transmitted blocks.
	
	> File I/O Abstraction
	  Used POSIX-based wrapper functions (myOpen, myRead, myWrite, etc.) to:
		- Read input files
		- Write encoded transmission data to ymodemSenderData.dat
		- Handle low-level system call errors robustly
	
	> Block Construction Logic
	  Completed and extended core functions in:
		- SenderY.cpp (e.g., genBlk, sendBlk, crc16ns)
		- PeerY.cpp (supporting communication logic)
	
	> Protocol Assumptions
		- Receiver always responds correctly (sends 'C' to initiate, ACKs all blocks)
		- No timing constraints or retransmissions required
		- Only 128-byte blocks used (no 1K blocks)

# Technical Challenges
	- Ensuring correct network byte order (endianness) for CRC computation across different architectures
	- Accurately replicating YMODEM control flow without an actual receiver
	- Implementing robust error handling for low-level file operations

# Outcome
	Produced a working sender simulation that generates a valid YMODEM byte stream for multiple files, suitable for later integration with a real or simulated receiver in subsequent project phases.


Testing & Validation:
	To ensure correctness and portability, I developed targeted test cases focused on validating byte ordering and error detection behavior across architectures.

# Endianness Verification
	
	> Designed tests to confirm correct handling of network byte order (big-endian) when computing and transmitting CRC values.
	> Simulated both little-endian and big-endian environments by:
		- Manually inspecting generated byte streams in ymodemSenderData.dat
		- Verifying that multi-byte fields (e.g., CRC output) are consistently encoded in big-endian format regardless of host architecture
	> Ensured compatibility with cross-platform systems where byte order differs

# CRC Validation

	> Created deterministic test inputs to validate the correctness of the CRC-16 implementation
	> Compared computed CRC values against known expected outputs
	> Verified that each generated block appends the correct CRC bytes in proper order

# Protocol Flow Testing

	> Simulated expected receiver behavior (ACK/NAK sequences) to validate:
		- Proper sequencing of blocks
		- Correct handling of End-of-Transmission (EOT) signaling
	> Confirmed that output file contents match the expected YMODEM transmission format

# Outcome
	These tests ensured that the implementation is architecture-independent, protocol-compliant, and robust against common data representation errors.
